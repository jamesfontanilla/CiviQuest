# UI/UX Design Principles

Apply these user experience principles when designing interfaces, interactions, and user flows.

## Visual Design & Perception

**Aesthetic-Usability Effect**
- Prioritize visual polish — users perceive attractive designs as more usable
- Invest in clean, modern aesthetics to build trust and credibility

**Law of Proximity**
- Group related elements close together
- Use whitespace to separate unrelated items

**Law of Similarity**
- Use consistent styling for similar elements (colors, shapes, sizes)
- Help users recognize patterns and relationships

**Law of Common Region**
- Use borders, backgrounds, or cards to group related content
- Create clear visual boundaries between sections

**Von Restorff Effect**
- Make important actions stand out through contrast
- Use distinctive styling for primary CTAs and critical information

## Decision Making & Choices

**Hick's Law**
- Minimize choices to speed up decision-making
- Break complex decisions into smaller steps
- Use progressive disclosure for advanced options

**Choice Overload**
- Limit options to prevent analysis paralysis
- Provide sensible defaults and recommendations
- Use filtering and categorization for large sets

**Occam's Razor**
- Choose the simplest solution that solves the problem
- Avoid unnecessary features and complexity

## Cognitive Load & Memory

**Miller's Law**
- Chunk information into groups of 5–9 items
- Use progressive disclosure to avoid overwhelming users
- Break long forms into logical sections

**Chunking**
- Group related information together
- Use formatting (spacing, headings) to create meaningful segments
- Apply to navigation, forms, content, and data displays

**Cognitive Load**
- Minimize mental effort required to use the interface
- Provide clear labels, helpful defaults, and contextual guidance
- Avoid jargon and ambiguous terminology

**Working Memory**
- Don't require users to remember information across screens
- Keep relevant context visible during multi-step processes
- Use inline validation and helpful error messages

## Interaction & Performance

**Fitts's Law**
- Make interactive targets large enough to click easily (min 44x44px for touch)
- Place frequently used actions close to where users need them
- Reduce distance users must travel with mouse or finger

**Doherty Threshold**
- Keep response times under 400ms for optimal productivity
- Provide immediate feedback for user actions
- Use loading indicators for operations over 1 second

**Goal-Gradient Effect**
- Show progress indicators for multi-step processes
- Provide positive reinforcement as users advance
- Make the finish line visible to maintain motivation

## Familiarity & Consistency

**Jakob's Law**
- Follow established conventions and patterns
- Users expect your interface to work like others they know
- Only innovate when it provides clear value

**Mental Models**
- Design to match users' existing understanding
- Use familiar metaphors and terminology
- Provide clear feedback that confirms their mental model

## Information Architecture

**Serial Position Effect**
- Place key actions at the beginning or end of lists
- Put most important navigation items first or last
- Structure content with strong openings and closings

**Postel's Law**
- Be flexible with user input (accept various formats)
- Be precise with output (provide clear, unambiguous information)
- Forgive user errors and offer helpful corrections

## Task Completion

**Zeigarnik Effect**
- Use progress indicators to show incomplete tasks
- Leverage notifications for unfinished workflows
- Help users resume interrupted tasks easily

**Parkinson's Law**
- Set reasonable time constraints for tasks
- Use deadlines to encourage completion
- Avoid open-ended processes without clear endpoints

**Peak-End Rule**
- Create positive moments during key interactions
- End experiences on a high note (success messages, celebrations)
- Pay special attention to onboarding and checkout flows

## Complexity Management

**Tesler's Law**
- Accept that some complexity is inherent
- Decide whether complexity lives in the system or with the user
- Automate and simplify where possible, but provide power when needed

**Pareto Principle**
- Focus on the 20% of features that deliver 80% of value
- Optimize common workflows over edge cases
- Prioritize based on user impact

## Attention & Focus

**Selective Attention**
- Design for focused attention on primary tasks
- Minimize distractions and unnecessary elements
- Use visual hierarchy to guide attention

**Flow**
- Remove friction from core workflows
- Provide clear next steps at each stage
- Balance challenge with user capability

## Implementation Guidelines

When building features:
- Start with user needs and mental models
- Apply relevant principles during design reviews
- Test with real users to validate assumptions
- Iterate based on behavioral data and feedback
- Document UX decisions and their rationale
